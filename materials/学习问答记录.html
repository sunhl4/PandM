条目 1

日期： 2026-03-26

问题：

（摘自 Phase1 01_量子计算基础与概念映射.ipynb）为什么一个自旋轨道要用一个量子比特？FCI 不应该对应的是整个量子电路吗？为什么说是对应量子态叠加？活性空间是什么，为什么对应逻辑量子比特？

回答：

（1）为什么一个自旋轨道对应一个量子比特？

这是在「占据数表象」下的一种编码约定，不是物理上说「一个电子就是一个比特」。

在二次量子化里，每个自旋轨道 \(j\) 上至多占据一个电子（泡利不相容），因此只有两个可能：空 或 占。这正好与量子比特的计算基 \(|0\rangle\)（空）、\(|1\rangle\)（占）一一对应。

\(M\) 个自旋轨道的一种电子组态（一个 Slater 行列式在占据数表象下）就对应一个积态：

\[ \lvert n_1 n_2 \cdots n_M \rangle = \lvert n_1\rangle \otimes \lvert n_2\rangle \otimes \cdots \otimes \lvert n_M\rangle,\quad n_j \in \{0,1\}. \]

注意：若用 Jordan–Wigner 等映射把费米子算符写成泡利串，一个费米子模式仍对应一条「占据」自由度，所以常说「自旋轨道数 = 量子比特数」（在未做对称性约化之前）。更精细时还会用 parity / tapering 减少比特数，那是后话。

（2）FCI 对应「整个电路」还是「叠加态」？你的直觉对了一半。

  • FCI（full CI） 指在给定单粒子基（给定一组自旋轨道）下，在所有满足粒子数与自旋约束的行列式张成的空间里精确对角化 \(\hat{H}\)，得到精确基态（在该基下）。
  • 它描述的是目标波函数（或等价地：在该子空间里哈密顿量的精确解），不是「某一条固定的量子电路」本身。

量子电路的作用是：制备某个态 \(\lvert\psi\rangle\)（例如 VQE 里参数化电路 \(U(\theta)\lvert\mathrm{ref}\rangle\)），或实现演化、测量等。

FCI 基态在占据数基下一般不是单个计算基矢（单个行列式），而是许多组态的线性叠加：

\[ \lvert\psi_{\mathrm{FCI}}\rangle = \sum_k c_k \lvert D_k\rangle, \]

其中每个 \(\lvert D_k\rangle\) 对应一个比特串（如 \(\lvert0110\cdots\rangle\)）。在量子语言里，这就是对多个计算基态的叠加。

所以笔记里写「FCI 波函数 \(\leftrightarrow\) 叠加态」指的是：FCI 所代表的态矢量，在量子比特计算基下通常表现为叠加；并不是说「FCI = 整条电路」。电路是制备/近似该态的手段（例如用 VQE 逼近这个叠加）。

（3）活性空间是什么？为什么说它对应「逻辑量子比特」？

活性空间（active space） 是量子化学里为了控制成本、集中描述强关联区域而选出的一部分空间轨道 + 在其中显式处理的电子。常见记号 CAS(\(n\), \(m\))：\(n\) 个活性电子在 \(m\) 条活性空间轨道里做完全组态相互作用（或 CASSCF/CASPT2 等）。

  • 冻芯 / 冻结占据：能量很低、化学上少参与成键的轨道，常固定为全占；
  • 截断虚轨道：高能虚轨道先丢掉，认为对目标性质不重要；
  • 活性部分：金属 d、反应键相关的 \(\sigma/\sigma^{\ast}\)、近前线轨道等，必须多组态/高阶关联才准。

在量子算法里，只有活性子空间会被映射成量子比特上的哈密顿量；比特数由「活性自旋轨道数」决定（例如 \(m\) 条空间轨道 \(\to\) 通常 \(2m\) 个自旋轨道 \(\to\) JW 下最多 \(2m\) 个量子比特，对称性约化后更少）。

这里说的「逻辑量子比特」在化学文献/算法讨论里，多指「这个量子化学问题在算法里用到的、有信息的量子比特个数」（问题规模），不要与量子纠错里的 logical qubit（1 个逻辑比特 = 很多物理比特）混淆。若上下文是容错量子计算，会单独讨论纠错开销；在 Phase1 笔记里，把它理解成算法希尔伯特空间的维度由活性空间大小决定即可。

（4）为何「活性部分」常指金属 d、反应键的 \(\sigma/\sigma^{\ast}\)、近前线轨道，且须多组态/高阶关联才准？

单参考（一个 Hartree–Fock 行列式）何时失效： HF 假设基态主要由一个 Slater 行列式支配。若多个电子组态能量接近、强烈混合，真实波函数形如 \(|\Psi\rangle \approx c_1|\Phi_1\rangle + c_2|\Phi_2\rangle + \cdots\)，单靠一个 \(|\Phi_0\rangle\) 误差很大，即存在显著 静态相关（static / nondynamical correlation）。多组态方法（如 CASSCF、活性空间内的 FCI）在选定轨道里显式叠加多个组态，正是为捕捉这一块。高阶关联（如 CASPT2、NEVPT2，或在合适参考下的耦合簇等）则主要补 动态相关——电子短程瞬时回避；单组态低阶截断往往不够。

  • 金属 d： 配体场下 d 分裂后，多种占据方式或 d–配体电荷转移组态可近简并；开壳层、不同自旋态/氧化态能量近时，单行列式难以公平描述。M–L 键共价成分强时，d 与配体 σ/π 强耦合，一个 HF 参考可能选错主导组态。故过渡金属常采用 CASSCF（含 d 与关键配体轨道）+ CASPT2/NEVPT2 一类流程。

  • 反应键相关的 \(\sigma/\sigma^{\ast}\)： 均裂或强极化断键时，成键阶段电子主要在 σ，拉长过程中 σ 与 σ* 占据图案剧烈重组，常出现多个行列式几乎简并（键断裂的多参考「教科书」情形）。单参考 HF 易给出错误的势能面形状（非物理尖点、能垒偏差）。沿反应路径需 多组态或 CASSCF 等使 PES 连续、合理。

  • 近前线轨道（HOMO、LUMO 及其邻近，及与之相关的低激发）： 能隙小则与单激发组态的能量分母小，低阶截断或单参考 CI 不稳定；基态中可混有显著 双激发/多组态 成分。单参考 TDDFT/CIS 在电荷转移、双激发、简并/锥交叉附近常失效；光谱与光化学常需 多参考（如 SA-CASSCF + CASPT2、MRCI 等）。

与上文「活性空间」的关系：FCI 在全基下精确但太大；活性空间把最易强相关、又决定键能/能垒/激发的那一段希尔伯特空间切出来做 FCI/CASSCF，其余用冻结核与微扰等处理。量子算法里映射到量子比特的，正是这类活性自旋轨道；选哪些轨道进活性空间，在化学上就由 d、σ–σ*、近前线 的角色决定。

（以上与 01_量子计算基础与概念映射.ipynb 中的「概念双语字典」一致，只是把电路 / FCI / 活性空间三者的分工说细了。）

条目 2

日期： 2026-03-27

问题：

为什么任意二阶厄米矩阵都可以展开成泡利算符串的张量积？

回答：

先把说法对齐一下：

  • 「二阶」在教材里常指 \(2\times 2\) 矩阵（单量子比特算符）。
  • 「泡利算符串」一般指 \(n\) 个量子比特上形如 \(\sigma_{i_1}\otimes\sigma_{i_2}\otimes\cdots\otimes\sigma_{i_n}\) 的算符，其中每个 \(\sigma_{i_k}\in\{I,X,Y,Z\}\)（有的书用 \(\sigma_0=I,\sigma_1=X,\ldots\)）。

量子化学 / VQE 里真正常用的是 \(2^n\) 维（\(n\) 个量子比特）的厄米算符分解；\(2\times 2\) 是它的 \(n=1\) 特例。下面两层一起说明。

（1）\(2\times 2\)：为什么任意厄米矩阵都能写成 \(I,X,Y,Z\) 的线性组合？

记泡利矩阵为 \(I,X,Y,Z\)（均厄米）。四个矩阵在 Hilbert–Schmidt 内积

\[ \langle A,B\rangle_{\mathrm{HS}} = \mathrm{Tr}(A^\dagger B) \]

下 两两正交（相差常数因子）：例如 \(\mathrm{Tr}(X)=\mathrm{Tr}(Y)=\mathrm{Tr}(Z)=0\)，且 \(\mathrm{Tr}(XY)=\mathrm{Tr}(XZ)=\cdots=0\)，而 \(\mathrm{Tr}(I^2)=2\)，\(\mathrm{Tr}(X^2)=\mathrm{Tr}(Y^2)=\mathrm{Tr}(Z^2)=2\)。

全体 \(2\times 2\) 复矩阵构成的向量空间维数是 \(2^2=4\)；线性无关的 \(I,X,Y,Z\) 共 4 个，因此 张成整个 \(M_2(\mathbb{C})\)。

任一 \(2\times 2\) 厄米矩阵 \(H\) 可唯一写成

\[ H = c_0 I + c_1 X + c_2 Y + c_3 Z,\quad c_k\in\mathbb{R}, \]

系数可用投影得到，例如 \(c_0 = \frac{1}{2}\mathrm{Tr}(H)\)，\(c_1 = \frac{1}{2}\mathrm{Tr}(HX)\) 等（前面的 \(\frac{1}{2}\) 来自归一化）。厄米保证这些 \(\mathrm{Tr}(H\sigma)\) 都是 实数，故 \(c_k\) 为实。

（2）\(n\) 量子比特：为什么任意 \(2^n\times 2^n\) 厄米矩阵都能写成泡利串的实系数线性组合？

令

\[ \mathcal{P}_n = \{\sigma_{i_1}\otimes\cdots\otimes\sigma_{i_n} \mid \sigma_{i_k}\in\{I,X,Y,Z\}\}. \]

  • 个数：每个位置 4 选 1，共 \(4^n\) 个串。
  • 全体 \(2^n\times 2^n\) 复矩阵的维数也是 \((2^n)^2 = 4^n\)。
  • 在 \(\langle A,B\rangle_{\mathrm{HS}}=\mathrm{Tr}(A^\dagger B)\) 下，不同泡利串 正交（可用张量积与 \(\mathrm{Tr}(A\otimes B)=\mathrm{Tr}(A)\mathrm{Tr}(B)\) 归纳验证）。因此这 \(4^n\) 个串 线性无关，从而构成 \(M_{2^n}(\mathbb{C})\) 的一组 基。

于是 任意（不必厄米）\(2^n\times 2^n\) 矩阵 \(M\) 都可写成

\[ M = \sum_{P\in\mathcal{P}_n} a_P\,P,\quad a_P\in\mathbb{C}, \]

其中 \(a_P = \mathrm{Tr}(PM)/\mathrm{Tr}(P^2)= \mathrm{Tr}(PM)/2^n\)（因每个泡利串平方为 \(I^{\otimes n}\)，迹为 \(2^n\)）。

若 \(M=H\) 厄米，每个 \(P\in\mathcal{P}_n\) 也 厄米（张量积保持厄米），则 \(a_P = 2^{-n}\mathrm{Tr}(HP)\) 为 实数。即

\[ H = \sum_{P\in\mathcal{P}_n} a_P\,P,\quad a_P\in\mathbb{R}. \]

这就是量子计算里把 哈密顿量写成泡利项之和 的线性代数依据：不是额外假设，而是 泡利串构成算符空间的正交基。

（3）小结

要点 内容
为什么能展开？ \(4^n\) 个泡利串在 HS 内积下正交且非零范数，张成 \(M_{2^n}(\mathbb{C})\)。
为什么厄米对应实系数？ 每个泡利串厄米，\(\mathrm{Tr}(HP)\) 为实。
与「张量积」的关系？ 每个基元本身就是 单比特泡利的张量积，线性组合仍是「泡利串的叠加」。

条目 3

日期： 2026-03-27

问题：

泡利展开会随着哈密顿算符变复杂而「指数」增加，尤其多电子体系；这种增加会体现在门电路变多。怎么解决这个问题？

回答：

先分清 两件事：(A) 形式上有多少个 可能的 泡利串；(B) 你手头这个 具体哈密顿量 实际出现多少 非零项，以及 电路/测量 上怎么付代价。

（1）「指数」在哪里、哪里其实常常是多项式？

  • 全集：\(n\) 个量子比特上，不同的泡利串最多 \(4^n\) 条——这是对 全体 \(2^n\times 2^n\) 矩阵作为线性空间维数 \((2^n)^2=4^n\) 的计数。若哈密顿量是 稠密、任意 的，最坏情况确实可能需要 \(\Theta(4^n)\) 项。

  • 多电子量子化学里常用的模型（二次量子化 + 给定单粒子基）：

\[ \hat{H} = E_0 + \sum_{pq} h_{pq}\,\hat{a}_p^\dagger \hat{a}_q + \frac{1}{2}\sum_{pqrs} h_{pqrs}\,\hat{a}_p^\dagger \hat{a}_q^\dagger \hat{a}_r \hat{a}_s. \]

其中 非零 \(h_{pq}\)、\(h_{pqrs}\) 的个数由 积分稀疏性 决定；对固定基组，两体项规模大致是 \(O(N^4)\)（\(N\) 为自旋轨道数），不是 \(4^N\)。

经 Jordan–Wigner、Bravyi–Kitaev 等映射后，每一项费米子算符会变成 若干条 泡利串的线性组合，但 总项数仍通常是 \(N\) 的多项式（常见量级 \(\sim O(N^4)\) 再乘上每个两体项映射出的常数个串）。

所以：「理论上基有 \(4^n\) 维」≠「你的化学哈密顿量一定有 \(4^n\) 项」。多电子体系让人头疼的，更多是 \(N\) 增大时 \(O(N^4)\) 仍爆炸，以及 比特数 \(n\propto N\) 带来的 深度、连通度、噪声 问题，而不一定是「纯 \(4^n\) 项」。

（1′）为何两体项规模是 \(O(N^4)\)，不是 \(4^N\)？

  • \(N\) 指什么： 固定基组下 自旋轨道（单粒子模式）条数；下标 \(p,q,r,s\) 各在 \(\{1,\ldots,N\}\) 里取值。

  • \(O(N^4)\) 从哪来： 两体部分是 \(\sum_{pqrs} h_{pqrs}\,\cdots\) —— 四个 求和下标，每个至多 \(N\) 种取值，故 可能的 \((p,q,r,s)\) 组合 粗计 \(N\times N\times N\times N\)。电子积分有 置换对称（如 \((pq|rs)\) 的八重等价），独立非零 积分个数会乘 常数因子（约 \(1/8\) 量级），但 随 \(N\) 渐近仍是 \(\Theta(N^4)\)：这是 「四个指标、每个 \(\sim N\)」 的组合计数，写作 \(N\) 的四次多项式。

  • \(4^N\) 是什么、为何混不进去： \(4^N\) 随 \(N\) 指数增长。它对应的是 另一类对象，例如 \(n\) 个量子比特 上 全体泡利串 的个数 \(4^n\)（条目 2–3：\((2^n)^2=4^n\) 维算符空间），这里常 \(n\propto N\)，故也可写成 \(4^{O(N)}\) 量级——但那是 「任意算符按泡利基展开时，基向量有多少个」，不是 「这一个 二次量子化哈密顿量里 有多少个非零 \(h_{pqrs}\)」。后者由 积分指标 决定，是 \(O(N^4)\)。

  • 记忆分辨： 数积分项 \(\Rightarrow\) 看 有几个下标在 \(1\cdots N\) 上跑（两体 \(\Rightarrow\) 四个下标 \(\Rightarrow\) \(N^4\)）；数泡利基大小 \(\Rightarrow\) 每个比特 4 种字母 \(\Rightarrow\) \(4^n\)。底数是 \(N\)、指数是 4（多项式）与 底数是 4、指数是 \(n\sim N\)（指数）不要对调。

（2）代价怎么体现在电路上？

常见有两条（可叠加）：

  1. 估计能量（VQE 等）：要算 \(\langle H\rangle = \sum_P a_P \langle P\rangle\)。每个 \(\langle P\rangle\) 要在 某组旋转 + 计算基测量 下得到；项很多 ⇒ 测量次数 / 电路套数 上升（不一定每项都单独一条极深线路，见下文 Pauli 分组）。

  2. 时间演化 / 相位估计：把 \(e^{-iHt}\) 分解成电路（乘积公式、LCU 等），项数与范数 进 深度或随机采样次数。

因此你的直觉对：体系变大 ⇒ 项变多 / 比特变多 ⇒ 实际量子资源（深度、测量、样本）恶化。只是恶化率对 标准电子哈密顿量 往往是 \(N\) 的多项式级别，外加常数因子与实现细节。

（3）「怎么解决」——没有单一魔法，是分层缓解

A. 从源头减少 \(N\) 与有效项数（化学侧）

  • 活性空间（CAS 等）：只保留关键轨道，直接降低 \(N\) 与 \(n\)。
  • 冻结核 / 截断虚轨道 / 积分阈值：让大量 \(h_{pqrs}\) 为 0 或可忽略，减少非零项。
  • 低秩、张量分解、密度拟合：在经典预处理阶段压缩两体部分，再映射到量子比特（减少有效相互作用项）。

B. 映射与对称性（量子比特侧）

  • Bravyi–Kitaev 等：在同样费米子算符下，有时比 JW 更浅的局域性（依问题而异）。
  • Tapering / 对称性约化：利用粒子数、自旋等对称性 减掉若干比特 或 合并等价自由度。
  • Contextual subspace、子空间方法：在更小的有效空间里构造哈密顿量。

C. 测量与估计（尽量不「一项一电路」）

  • Pauli 分组：将 两两对易 的泡利串放在 同一次测量（同一基旋转）里同时估，减少测量轮数。
  • 经典阴影（classical shadows）、derandomized / adaptive 测量：用更聪明的采样策略压低样本复杂度（各有假设与开销）。

D. 算法路线

  • ADAPT-VQE、启发式 ansatz：用 更少参数 / 更有针对性的幺正，避免盲目加深。
  • 量子子空间 / 压缩哈密顿量：在固定小子空间上对角化，降低有效规模。
  • 以经典为主、量子为辅：对某些块用 CCSD、DMRG 等，量子只处理 硬核关联子问题（混合方案）。

E. 若关心的是演化而非 VQE

  • qDRIFT、随机编译：用随机化减轻对 细项数 的线性依赖（换成样本数与精度权衡）。
  • 块编码、QSVT 等：在特定设定下改善 查询复杂度（实现门数仍要具体分析）。

（4）一句话收束

不能指望 对任意大分子仍用「全空间 + 全项 + 朴素单项测量」而资源不爆；工程上是用 化学截断 + 对称性约化 + 测量/算法设计 把 比特数、项数、电路深度、样本数 一起压下来。对标准二次量子化电子哈密顿量，泡利项数通常是 \(O(N^4)\) 量级，首要矛盾往往是 活性空间选多大 与 测量/编译如何优化，而不是误以为一定是 \(4^n\) 项全出现。

问题：

（Phase1 01_量子计算基础与概念映射.ipynb 门集表）T（π/8） 一行里写「\(R_z(\pi/2)\) 的固定角」——这句话怎么理解？

回答：

分三层：「固定角」指什么、和 \(R_z(\pi/2)\) 的准确关系、为何又叫 π/8 门。

（1）「固定角」相对谁而言？

在 NISQ / VQE 语境里，常见 连续可调 的单比特门 \(R_z(\theta)\)（\(\theta\) 是优化变量）。

T 门则是 角度写死 的离散门：

\[ T = \begin{pmatrix} 1 & 0 \\ 0 & e^{\mathrm{i}\pi/4} \end{pmatrix}, \]

不能像 VQE 那样把相位当成连续参数扫。容错量子计算里任意单比特旋转常 编译成 Clifford + T；T 的个数（T 深度 / T 计数）是资源度量之一。这里说的 固定角，就是指 非变分的、标准离散 Z 型相位门。

（2）与表中 \(R_z(\pi/2)\) 的对应（注意：不是矩阵逐元相等）

该笔记本采用的约定是

\[ R_z(\theta) = \begin{pmatrix} e^{-\mathrm{i}\theta/2} & 0 \\ 0 & e^{\mathrm{i}\theta/2} \end{pmatrix}. \]

于是 \(R_z(\pi/2)=\mathrm{diag}(e^{-\mathrm{i}\pi/4},\,e^{\mathrm{i}\pi/4})\)，与 S 门 \(S=\mathrm{diag}(1,\mathrm{i})\) 只差一个整体相位因子 \(e^{\mathrm{i}\pi/4}\)（单比特上即全局相位）。

另一方面 \(T^2=S\)（同样可差整体相位）。因此：

  • \(R_z(\pi/2)\) 在本表定义下描述的是 「相对相位转过 \(\pi/2\)」那一档 的 Z 型旋转，与 Clifford 的 S 同级；
  • T 是这一档的 平方根：再作用一次 T 才到 S / \(R_z(\pi/2)\) 那一级。

表上写「\(R_z(\pi/2)\) 的固定角」是 简略说法：不是 \(T=R_z(\pi/2)\)；而是 T 属于与 \(R_z(\pi/2)\)（即 S 档）同一族 Z 旋转的离散序列里、角度取在「一半」的那一步，且 角度不可连续调。

（3）为何文献里常叫 T 为「\(\pi/8\) 门」？

一种常见说法是：在 Bloch 球绕 \(z\) 轴 的记法里，有的教材把 旋转算符 写成 \(e^{-\mathrm{i}\alpha Z/2}\)，\(\alpha=\pi/4\) 时对应 \(|1\rangle\) 上多出的相位是 \(\pi/4\)（与上面的 \(T\) 一致），并把与 \(\alpha/2=\pi/8\) 相关的因子称作 \(\pi/8\) 门（命名随定义差一个「半角」因子，不同书符号不完全统一）。与 VQE 表里 的 \(R_z(\theta)=\exp(-\mathrm{i}\theta Z/2)\) 对照时，以 矩阵对角元 为准最不容易混。

一句话： 固定角 = 离散、非变分的标准相位门；与 \(R_z(\pi/2)\) 的关系 = 本表下 \(R_z(\pi/2)\) 与 S 同档，\(T^2=S\)，故 T 是 S / \(R_z(\pi/2)\) 那一级的平方根；π/8 = 文献对 \(\pi/4\) 相位门 的俗称，读不同定义时注意 半角。

日期： 2026-04-02

问题：

（Phase1 01_量子计算基础与概念映射.ipynb）CNOT：控制为 \(\lvert1\rangle\) 时对目标施加 X，控制为 \(\lvert0\rangle\) 时目标不变；可生成 Bell 态。Bell 态和 H 门有什么关系？ 请仔细说明。

回答：

结论先说清： 常用线路 \(H\) 打在控制比特上，再接 CNOT（控制→目标），从 \(\lvert00\rangle\) 得到 最大纠缠的 Bell 态 \(\lvert\Phi^+\rangle=\frac{1}{\sqrt{2}}(\lvert00\rangle+\lvert11\rangle)\)。H 负责把控制比特放进 \(\lvert0\rangle,\lvert1\rangle\) 的等权叠加；CNOT 把这种「控制取 0 或 1 的不确定性」翻译成两比特的量子关联。 仅有 H 或仅有 CNOT（从乘积初态出发）通常还不够得到 Bell 态。

（1）标准构造逐步算

初态 \(\lvert\psi_0\rangle=\lvert00\rangle\)（控制=第一比特，目标=第二比特，按 \(\lvert q_0 q_1\rangle\) 顺序）。

  1. \(H\otimes I\) 只作用在控制（第一比特）：

\[ \lvert\psi_1\rangle = (H\otimes I)\lvert00\rangle = \frac{1}{\sqrt{2}}(\lvert0\rangle+\lvert1\rangle)\otimes\lvert0\rangle = \frac{1}{\sqrt{2}}(\lvert00\rangle+\lvert10\rangle). \]

此时态仍是 乘积态（可写成「第一比特 ⊗ 第二比特」），纠缠度仍为 0。

  1. CNOT（控制第一比特，目标第二比特）：
    • 支路 \(\lvert00\rangle\)：控制为 \(\lvert0\rangle\)，目标 不 翻，得 \(\lvert00\rangle\)；
    • 支路 \(\lvert10\rangle\)：控制为 \(\lvert1\rangle\)，目标 X，\(\lvert10\rangle\to\lvert11\rangle\)。

\[ \lvert\psi_2\rangle = \mathrm{CNOT}\,\lvert\psi_1\rangle = \frac{1}{\sqrt{2}}(\lvert00\rangle+\lvert11\rangle)=\lvert\Phi^+\rangle. \]

可以验证 不存在 单比特态 \(\lvert\phi\rangle,\lvert\chi\rangle\) 使得 \(\lvert\Phi^+\rangle=\lvert\phi\rangle\otimes\lvert\chi\rangle\)，故为 纠缠态；对两比特而言与 完全混合约化态 对应的最大纠缠（Concurrence = 1 等）。

（2）H 和 CNOT 各自缺了会怎样？

  • 只有 CNOT、初态 \(\lvert00\rangle\)： 控制恒为 \(\lvert0\rangle\)，目标永不翻，\(\lvert00\rangle\to\lvert00\rangle\)，仍是平凡乘积态。
  • 只有 H、不要 CNOT： \(H\otimes I\) 作用在 \(\lvert00\rangle\) 上仍是 乘积态 \(\frac{1}{\sqrt{2}}(\lvert0\rangle+\lvert1\rangle)\otimes\lvert0\rangle\)，没有两比特关联。
  • 数学原因： 单个双比特门若形如 \(U\otimes I\) 或 \(I\otimes V\)（可分酉），把 乘积态 变成的仍是 乘积态。CNOT 不可写成 \(U\otimes V\)，才能 从某个乘积态 造出纠缠；但需要 合适的乘积态 作为输入——这里正是 H 在控制上造出的 \(\frac{1}{\sqrt{2}}(\lvert00\rangle+\lvert10\rangle)\)。

（3）直观图像

  • H：把控制从「确定是 \(\lvert0\rangle\)」变成「同时参与 \(\lvert0\rangle\) 与 \(\lvert1\rangle\) 两条支路」，且振幅相等。
  • CNOT：条件翻转——「若控制是 \(\lvert1\rangle\) 则目标翻」。于是在 \(\lvert1\rangle\) 支路上 \(\lvert0\rangle\to\lvert1\rangle\)，与 \(\lvert0\rangle\) 支路的「目标仍为 \(\lvert0\rangle\)」锁在一起，两比特 只能一起描述，不能拆开成各自一个态矢量。

（4）其它 Bell 态与 H / CNOT

四个标准 Bell 基可在 \(\lvert00\rangle\) 上先做单比特幺正再 CNOT 得到，例如：

  • \(\lvert\Phi^-\rangle=\frac{1}{\sqrt{2}}(\lvert00\rangle-\lvert11\rangle)\)：常在控制上加 \(Z\)（相对相位）后再接同样 \(H\)+CNOT。
  • \(\lvert\Psi^\pm\rangle=\frac{1}{\sqrt{2}}(\lvert01\rangle\pm\lvert10\rangle)\)：常把初态或目标侧换成 \(X\) 等再 \(H\)+CNOT。

H + CNOT 仍是核心 纠缠子线路；差别在 额外的 Pauli 相位/比特翻转。

一句话： H 并不「直接等于纠缠」，而是 在控制线上制备 \(\lvert0\rangle,\lvert1\rangle\) 的叠加；CNOT 利用 控制为 1 才翻目标 把两条支路 缝成 \(\frac{1}{\sqrt{2}}(\lvert00\rangle+\lvert11\rangle)\)。Bell 态 = 该组合的标准输出，故教材把 H（控制）+ CNOT 称为多比特线路里常用的 纠缠源。

条目 6

日期： 2026-04-02

问题：

（Phase1 01_量子计算基础与概念映射.ipynb §3.5）Gottesman–Knill：只有 Clifford + 计算基测量的线路，经典可高效模拟 → 单靠 Clifford 不够实现任意指数加速。Clifford + T：加上 T 后可近似通用量子计算；容错里 T 计数 由此而来。请详细解释。

回答：

先对齐 Clifford 是什么，再讲 Gottesman–Knill 到底「模拟了什么」，最后说明 为何要 +T 以及 T 计数 在容错里的含义。

（1）Clifford 门（与笔记本「泡利 → Clifford」一脉）

\(n\) 比特 Pauli 群 \(\mathcal{P}_n\) 由 Pauli 串 \(X\otimes Z\otimes\cdots\)（含相位）生成。幺正门 \(U\) 若对 任意 \(P\in\mathcal{P}_n\) 满足

\[ U P U^\dagger = \pm \mathrm{i}^k\, Q,\quad Q\in\mathcal{P}_n, \]

即 共轭下仍把 Pauli 映成 Pauli（允许整体相位），则 \(U\) 属于 Clifford 群。

常用生成元（与教材一致）：单比特 H、S（及 Pauli X,Y,Z）与双比特 CNOT 即可生成（在等价意义下）整个 Clifford 群。

直觉： Clifford 在 Pauli 观测量的共轭 下行为特别规整，因而可用 稳定子形式体系（stabilizer formalism） 紧凑描述一大类态与演化。

（2）Gottesman–Knill 定理在说什么？

定理（常用表述）： 若量子线路 仅由 Clifford 门 组成，初态为 计算基态（如 \(\lvert0\rangle^{\otimes n}\)），末端对 若干量子比特做计算基（Z）测量（可含经典随测量结果 条件 再做的 后续 Clifford，即 适应性 Clifford 线路 的适当版本仍常被包含在可模拟范围内——细节依模型，本科阶段可记主干结论），则 在经典计算机上可在多项式时间内 模拟 测量结果的采样分布（以及许多相关的 Pauli 期望值、稳定子更新等）。

要点不是「Clifford 线路没有纠缠」——CNOT + H 可以产生 Bell 态 等高度纠缠；要点是 这类纠缠落在 稳定子态（stabilizer state） 子空间里，其结构允许 经典跟踪（稳定子生成元随门 共轭更新，规模 \(\mathcal{O}(n)\) 比特描述）。

因此：仅 Clifford + 计算基测量 的模型，不包含 已知的 BQP-完全 那类「全量子」难度；教材里写 单靠 Clifford 不够实现任意指数加速，指的是：若你的算法世界只有 Clifford 线路这一种量子资源，经典端已有高效模拟途径，便 不能 指望它像含 非 Clifford 的通用量子线路那样，在复杂度假设下形成 对经典世界的根本性分离（与 Shor、某些采样问题等需 非 Clifford 的图景相对照）。

措辞注意： 「指数加速」在文献里有多层含义；此处与笔记本一致，强调 Clifford 子集的可经典模拟性 与 通用量子计算 之间的 能力鸿沟。

（3）为何要 Clifford + T？「近似通用」

Clifford 门单独不构成通用量子门集：不能仅靠 Clifford 任意逼近 任意 \(n\) 比特酉（例如 T 门 本身 不是 Clifford）。

Solovay–Kitaev 定理（思想）：给定一个 稠密 的由某有限门集生成的子群，任意目标单比特酉可用该门集 \(\varepsilon\)-逼近；Clifford + T 是实践中常用的 有限 集合，T（或等价地，任一 非 Clifford 单比特门）提供 走出稳定子世界 的「缺口」，使得生成的单比特（进而多比特）幺正 稠密，从而 近似通用（任意线路可用 Clifford+T 以可控误差逼近）。

因此笔记本写：加上 T（或任一非 Clifford 单比特门）后，可近似通用——这是 容错与编译 里把连续 \(R_z(\theta)\) 等 分解为 Clifford + T 的理论背景。

（4）容错里的 T 计数 / T 深度

在 容错量子计算 的标准图景里：Clifford 门（H、S、CNOT 等）常可用 容错 Clifford 子程序 相对「便宜」地实现；T 门（或非 Clifford）往往要通过 magic state distillation 等 额外资源 实现，空间、时间、噪声预算 常 主要由 T 的个数与并行深度主导。

因此论文与工程里常报告 T 深度（T-depth）、T 计数（T-count） 作为 线路代价 的指标——这正是笔记本「T 计数 的来源」一句的含义：通用性来自非 Clifford（如 T），而代价分析盯住 T。

（5）和 NISQ / VQE 的对比（笔记本「两套语言」）

NISQ / VQE 常直接写连续 \(R_y(\theta), R_z(\theta)\)；容错线路 则把同一幺正 编译 成 Clifford + T 序列，讨论 T 计数。同一条物理算法，只是 离散化与计费单位 不同；Gottesman–Knill 与 Clifford+T 主要在 容错、复杂性、模拟边界 语境下最重要。

一句话： Gottesman–Knill = Clifford（+适应测量）线路可在经典上 多项式模拟，故 单靠 Clifford 不能支撑「全量子」意义上的任意硬分离；+T 才 近似通用；T 计数 = 容错实现里 非 Clifford 成本 的常用度量。

问题：

弦算符和泡利串的关系？

回答：

以下与 Phase1 01_量子计算基础与概念映射.ipynb 中 Jordan–Wigner（JW） 一小节的用语一致：弦算符即 JW 里为保证费米统计而出现的那一串 \(Z\)；泡利串是 \(I,X,Y,Z\) 的张量积。二者关系见下；文末附带 产生/湮灭算符 与泡利串的对应，便于和哈密顿量映射衔接。

（1）弦算符是什么？

在 JW 映射里，第 \(j\) 个模式的产生算符写成

\[ \hat{a}_j^\dagger = \frac{1}{2}(X_j - iY_j)\otimes Z_{j-1}\otimes\cdots\otimes Z_0 \]

（湮灭算符 \(\hat{a}_j\) 为相应厄米共轭形式。）

其中 前缀

\[ Z_{j-1}\otimes\cdots\otimes Z_0 \]

（在有的排序约定里写在 \((X_j-iY_j)/2\) 的另一侧，本质相同）就叫 弦算符：一串作用在 更低指标比特 上的 \(Z\)，像一条「尾巴」。

（2）和泡利串的关系：弦算符本身就是泡利串的一部分

  • 泡利串（Pauli string）：\(n\) 个比特上 \(I,X,Y,Z\) 的张量积，例如 \(Z_0 Z_2 X_4\)（省略 \(\otimes\) 记号）。
  • 弦算符在 JW 里 不是 另一种矩阵代数对象，它就是 只含 \(Z\)（以及其余位置 \(I\)） 的那一段张量积。

因此：整个 \(\hat{a}_j^\dagger\)（或 \(\hat{a}_j\)）在量子比特上 不是 单条泡利串——因为 \((X_j-iY_j)/2\) 拆开后涉及 \(X_j\) 与 \(Y_j\)——但它是 少数几条泡利串的线性组合；而 反对易结构所依赖的 相位/符号，正是由这段 \(Z\) 弦 与前面的 \(X,Y\) 按泡利乘法规则乘出来保证的。

一句话：弦算符 = JW 里为保证费米统计而配的 \(Z\) 型泡利子串；完整的费米子算符 = 局域的 \(X/Y\)（及组合） ⊗ 这条 \(Z\) 弦（再叠加系数）。

（3）延伸：产生/湮灭算符与泡利串

\(\hat{a}_p^\dagger,\hat{a}_q\) 等本身不是泡利串；经 JW（或 Bravyi–Kitaev 等）后，每个费米子单项式变成 若干泡利串的线性组合（其中含 弦算符 对应的 \(Z\) 段）。哈密顿量里 \(\hat{a}^\dagger\hat{a}\)、\(\hat{a}^\dagger\hat{a}^\dagger\hat{a}\hat{a}\) 等乘开、合并后，最终得到程序里的 SparsePauliOp 那种「很多泡利串 + 实系数」。

（4）小结

对象 与泡利串的关系
弦算符（JW） \(Z\) 的张量积段，是某条（或多项展开里共同的）泡利子串。
\(\hat{a}_j^\dagger,\hat{a}_j\) 一般是 多条 泡利串的 线性组合（含 \(X_j,Y_j\) 与 \(Z\) 弦）。
电子哈密顿量 \(\hat{H}\) 费米子多项式 \(\to\) 许多泡利串之和（条目 2、3 讨论的分解）。

问题：

（01_量子计算基础与概念映射.ipynb）return jordan_wigner_creation(j, n_qubits).conj().T 里 .conj().T 是什么意思？

回答：

在 NumPy 里，对矩阵 A：

  • A.conj()：对每个元素取 复共轭 \(\overline{A_{ij}}\)。
  • .T：转置，\((A^{\mathsf T})_{ij}=A_{ji}\)。

先后做 共轭再转置，得到的就是数学上的 厄米共轭（伴随）

\[ A^\dagger = (A^{\mathsf T})^\ast, \]

在代码里常写成 A.conj().T（与 np.conjugate(A).T 同义）。

为什么湮灭算符要这么写？

费米子代数里 湮灭算符 与 产生算符 互为 厄米共轭：

\[ \hat{a}_j = \bigl(\hat{a}_j^\dagger\bigr)^\dagger. \]

jordan_wigner_creation 返回的是 \(\hat{a}_j^\dagger\) 在计算基下的 矩阵。矩阵表示下，伴随就是 .conj().T，所以

def jordan_wigner_annihilation(j, n_qubits):
    return jordan_wigner_creation(j, n_qubits).conj().T

就是在实现 \(\hat{a}_j = (\hat{a}_j^\dagger)^\dagger\)，无需把 JW 公式再抄一遍。

小注： 若矩阵全是实数，.conj() 不变，.conj().T 等价于只 转置；但一般 JW 里会出现 \(i\)（如 \(X-iY\)），所以保留 .conj().T 才始终正确。

（01_量子计算基础与概念映射.ipynb 中 decompose_to_pauli）为什么 abs(coeff) > 1e-12 就和「厄米矩阵系数为实数」有关？

对「几乎是实数」的 coeff，abs(coeff) 就是其 模；若实部、虚部都极小，整项丢弃。

np.real(coeff) 是 数值卫生：去掉应约为零的虚部，避免后续重构或打印时出现假「复数」。

代码 作用
abs(coeff) > 1e-12 稀疏化：忽略数值上可视为 0 的项。
np.real(coeff) 去浮点虚部：与厄米性保证的「实系数」配套使用。

问题：

（01_量子计算基础与概念映射.ipynb 里 H_simple）这个哈密顿量为什么是这个矩阵？随便写的还是真有物理对应？

（1）它在数学上是什么？

\[ H_{\mathrm{simple}} = \begin{pmatrix}-1.0 & -0.5\\ -0.5 & 1.0\end{pmatrix} \]

厄米、迹为 0，可看成 两能级系统 的有效哈密顿量：对角元像两个「在位」能量（一低一高），非对角像两态之间的 耦合/跃迁（实数表示厄米、时间反演下常见的选择）。这类形式在双阱、二能级有效模型里很常见，但 \(-1,-0.5,1\) 这组具体数字 在本格代码里 没有 再往下推到「某个 \(U,t,\varepsilon\)」的固定公式——属于 示意数量级。

（2）和「单轨道 / Hubbard」字样怎么对齐？

  • 标准单格点、两自旋 的 Hubbard 模型，占据数表象下希尔伯特空间维数是 \(4\)（空、↑、↓、双占），哈密顿量一般是 \(4\times 4\)，含 在位能 \(\varepsilon\)、库仑 \(U\)、有时还有杂化 等，不会自然等同于本例这一个 \(2\times 2\) 矩阵。

原注释已改为：演示用任意厄米矩阵 + 与两能级图像的粗略类比，并提醒单格点两自旋 Hubbard 常为 \(4\times 4\)。

（3）若你想要「真有对应」的例子可以怎么做？

  • 从 具体二体系（如 H\(_2\) 在极小基下经对称性约化后的 \(2\) 量子比特块）用 Qiskit Nature / PySCF 读出 数值矩阵，再分解泡利；
  • 或自己给定 \(t,U,\varepsilon\)，在 选定的子空间维数 下写出 Hubbard 的矩阵，再与这里对照。

条目 11

日期： 2026-04-02

问题：

（Phase1 01_量子计算基础与概念映射.ipynb，泡利分解代码格注释）「可粗略类比 两能级 + 耦合：对角像在位能，非对角像跃迁/杂化；标准单格点 Hubbard（两自旋）通常是 4×4」——分别是什么意思？

回答：

这一行是在说明：示例里的 2×2 厄米矩阵 在物理上可以怎么想；并对比 为什么单格点 Hubbard 会变成 4×4（与「一个量子比特 = 2×2」不是同一套希尔伯特空间）。

（1）2×2 厄米矩阵 = 「两能级」模型

选定 两个正交归一基矢（记作 \(\lvert\phi_0\rangle,\lvert\phi_1\rangle\)，量子计算里常直接记 \(\lvert0\rangle,\lvert1\rangle\)），任意 2×2 厄米矩阵 \(H\) 就是 \(\{\lvert\phi_0\rangle,\lvert\phi_1\rangle\}\) 张成的 二维子空间 上哈密顿量的矩阵元：

\[ H_{ij}=\langle\phi_i\lvert \hat{H}\rvert\phi_j\rangle,\quad i,j\in\{0,1\}. \]

因此它描述的是 只有两个相关能级（或两种组态） 的有效问题——与 单量子比特 的 \(2\) 维希尔伯特空间 形式上一致，故可用 \(I,X,Y,Z\) 做泡利分解（代码里做的事）。

（2）对角元 ≈ 「在位能」

对角元 \(H_{00}\)、\(H_{11}\) 是 各自基矢上的能量期望值（在该基下）。若暂时 忽略非对角（令 \(H_{01}=0\)），则 \(\lvert\phi_0\rangle\) 与 \(\lvert\phi_1\rangle\) 不混合，能量就是 \(H_{00}\) 与 \(H_{11}\)，像两个 彼此独立 的「位点」或 两种组态各自的在位能。

化学直觉例子：两 价键/离子 极限结构作为基，对角元对应 各结构能量；或格点模型里 电子已在某一侧 的能量。

（3）非对角元 ≈ 「跃迁 / 杂化」

非对角 \(H_{01}=H_{10}^\ast\) 不为零 时，\(\lvert\phi_0\rangle\) 与 \(\lvert\phi_1\rangle\) 直接耦合：\(\hat{H}\) 的本征态一般是 两者的线性组合（杂化），时间演化下两振幅会 交换权重（跃迁、隧穿、hopping 等说法，取决于你的基在物理上代表什么）。

笔记本中的示例

\[ H_{\mathrm{simple}}=\begin{pmatrix}-1.0&-0.5\\-0.5&1.0\end{pmatrix} \]

即有 对角能量偏置（一态偏低、一态偏高）又有 非对角耦合，是典型的 两能级 + 耦合 玩具哈密顿量；分解到泡利后会出现 \(Z\)（对角差） 与 \(X\)（实非对角） 等项，和 横场 Ising / 两能级驱动 同一类图像。

（4）为何「单格点、两自旋」Hubbard 是 4×4？

一个空间格点 上通常有 两条费米子模式：自旋 ↑ 与 ↓ 各一条（两条单粒子态）。在 占据数表象 下，每条模式 空或占，共 \(2\times2=4\) 种组态：

\[ \lvert\varnothing\rangle,\quad c^\dagger_{\uparrow}\lvert\varnothing\rangle,\quad c^\dagger_{\downarrow}\lvert\varnothing\rangle,\quad c^\dagger_{\uparrow}c^\dagger_{\downarrow}\lvert\varnothing\rangle \]

（常简记为 空、单占↑、单占↓、双占）。在这 四维 Fock 子空间 里，\(\hat{H}\) 是 4×4 矩阵。

Hubbard 相互作用 \(U\,\hat{n}_\uparrow\hat{n}_\downarrow\) 等会 特别改变「双占」态的能量，故 不能 在不加近似的情况下压成 单个 2×2 问题——除非你再做 投影 / 有效模型 只保留其中 两个 态（例如低能两态模型），那时才会又出现 2×2。

（5）和本格代码的关系

  • 代码里的 2×2：演示 单量子比特哈密顿量 的 泡利展开；注释用 两能级 + 耦合 帮你和 量子化学 / 固体 语言对齐。
  • 4×4 Hubbard：提醒 费米子两模式 的 完整小块 已是 4 维；多格点 则维数 指数增长，才需要 JW 映射到多量子比特、或 活性空间截断 等。

一句话： 对角 = 两基矢各自的「在位」能量标度；非对角 = 两基矢 混合/跃迁；单格点两自旋 Hubbard = 4 个占据组态 ⇒ 4×4；笔记本 2×2 是 两能级有效模型 / 单比特 的演示，维数不要和 4×4 混为一谈。

问题：

\[ H_{H_2} = g_0 I + g_1 Z_0 + g_2 Z_1 + g_3 Z_0 Z_1 + g_4 Y_0 Y_1 + g_5 X_0 X_1 \]

（Phase1 01_量子计算基础与概念映射.ipynb §5）这公式的推导过程是什么？

回答：

这条式子 不是 从第一性原理「凭空猜」出来的，而是下面链条的 结果：（1）固定分子与基组，写出 二次量子化 电子哈密顿量；（2）用 Jordan–Wigner（或等价费米子–量子比特映射） 把 \(\hat{a},\hat{a}^\dagger\) 换成泡利算符；（3）利用 对称性 / 子空间编码 把 STO–3G 下 4 个自旋轨道 的问题 约化到 2 个量子比特；（4）在 2 比特空间里，一般本可出现 最多 16 项 泡利基，但 H\(_2\) 的对称性使 只有 6 类项系数非零；\(g_i\) 由 单、双电子积分（或等价地，由约化后的 \(4\times 4\) 矩阵）算出来。笔记本里系数来自 Peruzzo et al. (2014) 一文常用的 STO–3G、平衡键长数据。

（1）出发点：二次量子化哈密顿量

在固定单粒子基（自旋轨道指标 \(p,q,r,s\)）下，

\[ \hat{H} = \sum_{pq} h_{pq}\,\hat{a}_p^\dagger \hat{a}_q + \frac{1}{2}\sum_{pqrs} h_{pqrs}\,\hat{a}_p^\dagger \hat{a}_q^\dagger \hat{a}_r \hat{a}_s + E_{\mathrm{nuc}}, \]

其中 \(h_{pq}\)、\(h_{pqrs}\) 由 量子化学积分（重叠、动能、核势、电子排斥）在选定基组下算出；H\(_2\) + STO–3G 时 4 条自旋轨道，故指标跑在 \(\{0,1,2,3\}\)。

（2）Jordan–Wigner：把每一项费米子算符变成泡利串

例如第 \(j\) 个模式（笔记本记号）

\[ \hat{a}_j^\dagger = \frac{1}{2}(X_j - iY_j)\otimes Z_{j-1}\otimes\cdots\otimes Z_0,\quad \hat{a}_j = (\hat{a}_j^\dagger)^\dagger. \]

将每个 \(\hat{a}_p^\dagger \hat{a}_q\)、\(\hat{a}_p^\dagger \hat{a}_q^\dagger \hat{a}_r \hat{a}_s\) 代入 上式，用 泡利代数（\(X^2=Y^2=Z^2=I\)、\(XY=iZ\) 等）乘开、合并同类泡利串。

  • 单体项 \(\hat{a}_p^\dagger \hat{a}_q\)：JW 后得到 若干泡利串，典型会同时出现 \(Z\) 弦 与局域 \(X,Y\)（见条目 7）。
  • 双体项 \(\hat{a}_p^\dagger \hat{a}_q^\dagger \hat{a}_r \hat{a}_s\)：展开后项数更多，但仍是 泡利串的线性组合，系数由 \(h_{pqrs}\) 与 \((X\pm iY)\) 展开后的 实部、虚部 组合决定。

此步完成后，你得到的是 4 量子比特 上的 长泡利展开（很多项），还不是题目里的 2 比特 6 项。

（3）为什么能写成「只有 2 个量子比特」？

STO–3G 的 H\(_2\) 有 2 电子；全希尔伯特空间 \(2^4=16\) 维，但 物理哈密顿量与粒子数算符对易，关心基态时常在 \(N=2\) 子空间（维数 \(\binom{4}{2}=6\)）内工作。

进一步，H\(_2\) 有 额外对称性（如自旋、部分费米子宇称等，依映射而定），可用 tapering / 固定量子数编码（文献与 Qiskit Nature 里常称对称性约化）消去冗余比特，把有效自由度压到 2 个量子比特（\(2^2=4\) 维基与某个选定的 4 维子空间或等价编码对应——具体编码依论文或 qiskit_nature 选项而定）。

要点：式中的 \(Z_0,X_0\) 等是 约化后 2 比特问题 的泡利算符，不一定 等于「未约化时第 0 号自旋轨道那一格」的同名算符；笔记本括号里写的 「粒子守恒约化到 2 量子比特」 指的就是这一步 编码/约化，而不是 JW 本身。

（4）为什么偏偏是 \(I,Z_0,Z_1,Z_0Z_1,X_0X_1,Y_0Y_1\) 这六类？

一般 2 量子比特 厄米算符可写成 16 个 \(P_0\otimes P_1\)（\(P_k\in\{I,X,Y,Z\}\)）的 实系数 线性组合（条目 2 的 \(n=2\) 特例）。

H\(_2\) 在 实数积分、分子对称性（\(D_{\infty h}\) 等）及所选 2 比特编码 下，许多泡利项系数 严格为 0，常见 存活项 的结构就是：常数项、局域 \(Z\)、\(ZZ\)、以及 \(XX\) 与 \(YY\) 成对出现且系数常相等（你笔记本代码里 XX 与 YY 同为 0.1807）。

  • \(Z\)、\(ZZ\)：多来自 占据数 / 库仑–交换 的对角结构在编码后的表现。
  • \(XX\)、\(YY\)：多来自 单激发型 费米项 \(\hat{a}_p^\dagger \hat{a}_q + \mathrm{h.c.}\) 经 JW 后 \(X,Y\) 组合；在 H\(_2\) 这种 实哈密顿量 + 对称性 下，往往以 \(X_0X_1\) 与 \(Y_0Y_1\) 的特定组合出现，且可出现 \(g_4=g_5\)。

若编码或基组不同，非零项集合 可能略有变化；你引用的 6 项形式 是 该文献/该约化路径 下的标准写法。

（5）系数 \(g_i\) 怎么从「推导」落到数字？

两条等价路线（实务上常混用）：

  1. 化学软件：用 PySCF、Psi4 等算积分 → qiskit_nature.second_q 建费米子算符 → JordanWignerMapper（及对称性约化/taper）→ 得到 SparsePauliOp，读出各 \(g_i\)。
  2. 线性代数：若已有约化后 \(4\times 4\) 厄米矩阵 \(H\)（某组计算基下），则每个泡利串 \(P\) 的系数为 Hilbert–Schmidt 投影（2 比特下 \(\mathrm{Tr}(P^2)=4\)）：

\[ g_P = \frac{1}{4}\,\mathrm{Tr}(P\,H). \]

这与笔记本里单比特 decompose_to_pauli 里 \(\mathrm{Tr}(PH)/2\) 是同一逻辑的推广（\(n=1\) 时 \(\mathrm{Tr}(P^2)=2\)）。

（6）小结（推导链一句话）

积分 \(\to\) 二次量子化 \(\hat{H}\) \(\to\) JW（+Pauli 代数化简）\(\to\) 对称性约化到 2 比特 \(\to\) 非零泡利类只剩 6 类 \(\to\) \(g_i\) 由积分或 \(\mathrm{Tr}(PH)/4\) 确定。

若要 自己重算一遍 \(g_i\)，建议在工程里用 Qiskit Nature 从 H\(_2\) 直接 build 哈密顿量 与笔记本中 Peruzzo 系数对照，比手算所有 JW 展开更可靠。

问题：

（Phase1 01_量子计算基础与概念映射.ipynb §5 H₂ 示例）流程是：积分 → 二次量子化 \(\hat{H}\) → JW（+Pauli 化简）→ 对称性约化到 2 比特 → 非零泡利只剩 6 类 → \(g_i\) 由积分或 \(\mathrm{Tr}(PH)/4\) 定。希望 更细 理解：对称性约化 在做什么、为什么只剩 6 项、\(\mathrm{Tr}(PH)/4\) 怎么用。

回答：

下面按 链条 讲；6 项 与 \(\mathrm{Tr}(PH)/4\) 放在 已经得到 2 比特矩阵 \(H\) 之后 最清楚。

（1）积分 → 二次量子化

在 给定基组（如 H₂ STO-3G）下，先做 HF 得 分子轨道，再算 单电子积分 \(h_{pq}\)、双电子积分（\((pq|rs)\) 等，记号依文献）。它们进入

\[ \hat{H} = E_{\mathrm{nuc}} + \sum_{pq} h_{pq}\,\hat{a}_p^\dagger \hat{a}_q + \frac{1}{2}\sum_{pqrs} h_{pqrs}\,\hat{a}_p^\dagger \hat{a}_q^\dagger \hat{a}_r \hat{a}_s \]

（具体 \(h_{pqrs}\) 与 \((pq|rs)\) 的对应关系按你用的 chemist / physicist 记号 换序，这里不展开。）到这一步，\(\hat{H}\) 仍是 费米子算符，作用在 Fock 空间 上。

（2）JW：四个自旋轨道 → 先得到「4 量子比特」表述

H₂ STO-3G：2 个空间轨道 × 2 自旋 = 4 条自旋轨道，每条对应 一个 JW 费米子模式，最直白 的映射是 \(n=4\) 个量子比特。

把每个 \(\hat{a}_p^{(\dagger)}\) 换成 Pauli 串（带 Jordan–Wigner 弦 \(Z_0 Z_1\cdots Z_{p-1}\)），再把 多项乘开，用 \(XY=\mathrm{i}Z\) 等 Pauli 代数 合并同类项，得到 仅含 \(I,X,Y,Z\) 张量积 的哈密顿量。

注意： 在 未约化 的 4 比特形式下，泡利展开里 远多于 6 项；笔记本里的 6 项 是 约化之后、作用在 2 比特子空间上的 \(4\times 4\) 有效哈密顿量 的展开。

（3）对称性约化：从 4 比特「压」到 2 比特——在干什么？

目标： 真实物理的 H₂（固定 电子数、关心 基态 所在的小子空间）不需要 在 全部 \(2^4=16\) 维计算基上写最一般的 \(\hat{H}\)；利用 与 \(\hat{H}\) 对易的量子数，可把问题 限制在更小的有效空间，并 减少比特数。

对分子哈密顿量（JW 之后），常用思路包括：

  • 费米子层面的对称性： 粒子数 \(\hat{N}\)、自旋投影 \(\hat{S}_z\)（闭壳 / 单态 sector）、点群（H₂ 有 反演 \(g/u\) 等）。\(\hat{H}\) 与这些算符 对易 ⇒ 在 共同本征子空间 里块对角，其它块与基态无关 时可丢掉。
  • 量子比特层面的「削比特」（tapering，思想）： 若存在 Pauli 型 对称算符 \(G\)（由 JW 后的对称性推出），满足 \([H,G]=0\) 且 \(G^2=I\)，则整个希尔伯特空间按 \(G\) 的本征值 \(\pm 1\) 分成两块；在 固定本征值 的扇区里，\(G\) 相当于 \(\pm I\)，可用 经典比特记录该量子数，从而 消去一个依赖比特——多个独立对称可 多次 削，4 比特 → 2 比特 正是 消掉两个这类自由度 的典型结果（具体选哪两个对称，由实现与文献约定决定；笔记本概括为 粒子守恒等对称性）。

直觉： 不是「随便删掉两个比特」，而是 「基态（及 VQE 关心的态）只活在一个 4 维子空间里，且该子空间可用 2 个量子比特标记」；其余 2 个比特上的信息被 对称性固定，不必带进量子处理器。

（4）为什么约化后「非零泡利类」常只剩 6 类（II, IZ, ZI, ZZ, XX, YY）？

在 已经得到的 \(4\times 4\) 有效厄米矩阵 \(H\)（2 比特）上，原则上 可展成 16 个 Pauli 串的基：

\[ H = \sum_{P\in\{I,X,Y,Z\}^{\otimes 2}} c_P\,P. \]

6 项 的意思是：其中 10 个 \(c_P=0\)。原因来自 有效哈密顿量的结构，而不是「Pauli 基只有 6 个」：

  • 实厄米 + H₂ 最小基集下的积分选择定则： 许多 单比特 \(X\)、\(Y\) 单独出现 或 \(X\) 与 \(Z\) 混排 等项，对应 在分子轨道对称性下为零的矩阵元（或整块在选定的 单态 / \(S_z=0\) 子空间里 矩阵元恰为零）。
  • 常见模式： 保留 常数项 + 局域 \(Z\) 型（II, IZ, ZI, ZZ，来自占据数 / 库仑交换的对角结构在 Pauli 表象下的分解），以及 \(\frac{1}{2}(X_0 X_1 + Y_0 Y_1)\) 型 配对项（与 单态 下 \(\lvert01\rangle\leftrightarrow\lvert10\rangle\) 的 自旋翻转激发 有关）；IX, IY, XZ, … 等在该 \(4\times4\) 块里 系数算出来就是 0。

因此：6 = 这具体问题里非零系数个数；换分子、换基组、换是否再削对称，项数会变。

（5）\(g_i\) 与 \(\mathrm{Tr}(PH)/4\)：怎么算、和积分什么关系？

  • 两比特 上，Pauli 串 \(P\) 满足 \(\mathrm{Tr}(P^2)=\mathrm{Tr}(I)=4\)，且不同串 Hilbert–Schmidt 正交：\(\mathrm{Tr}(PQ)=0\)（\(P\neq Q\)）。因此

\[ c_P = \frac{\mathrm{Tr}(P\,H)}{\mathrm{Tr}(P^2)} = \frac{\mathrm{Tr}(P\,H)}{4}. \]

这就是笔记本里 \(\mathrm{Tr}(PH)/4\)（记号与 \(P\) 顺序无关）。只要你手里已经有 与线路一致的 \(4\times4\) 矩阵 \(H\)，逐个 \(P\in\{II,IX,\ldots,ZZ\}\) 做 迹 就得到所有系数；其中 10 个为 0 就得到 6 个非零 \(g_i\)。

  • 和积分的关系： 实务中 顺序是反过来的：用 量子化学程序 由 积分 组装 费米子 \(\hat{H}\) → JW + 对称性程序 得到 数值矩阵 \(H\)（或等价泡利分解）。Tr 公式 是 从最终矩阵读回 Pauli 系数 的 线性代数定义，与「积分怎么来的」不矛盾——同一 \(H\)，从积分走 与 先建矩阵再 Tr 必须一致（数值误差除外）。

（6）小结表

阶段 对象 维数 / 项数直觉
积分 \(h_{pq},(pq\|rs)\) 等 两体 \(\sim O(N^4)\)
二次量子化 \(\hat{H}\) 费米子 Fock 空间很大
JW（未削） Pauli 串，\(n=4\) 比特数 4；展开项多
对称性约化 有效 \(H\) 在 2 比特 \(4\times4\) 矩阵
Pauli 展开 \(c_P=\mathrm{Tr}(PH)/4\) 最多 16 项，本例 6 项非零

一句话： 对称性约化 = 用 与 \(H\) 对易的量子数 把问题 缩到小希尔伯特空间并少用量子比特；6 项 = 该 2 比特有效 \(H\) 在 16 个 Pauli 基下 只有 6 个非零系数；\(\mathrm{Tr}(PH)/4\) = 这 6 个（及隐藏的 10 个零）系数的统一定义。

活性空间 / CI 截断 是 在哈密顿量或基组层面 扔掉某些组态；这里的 0.01 只是 print，不参与 能量计算——本征矢仍是 完整 4 维（或 \(2^n\) 维） 上算出来的 精确（在该矩阵意义下）结果。

一句话： abs(coeff) > 0.01 = 打印时只显示「够大」的分量，方便看 主导组态 并 过滤数值毛刺；不是 物理近似。

条目 16

日期： 2026-03-27

问题：

（Phase1 01_量子计算基础与概念映射.ipynb VQE 演示）参数化态

\[ \lvert\psi(\theta)\rangle = \cos\frac{\theta}{2}\lvert\mathrm{HF}\rangle + \sin\frac{\theta}{2}\lvert\mathrm{double\ exc}\rangle \]

为什么用 cos、sin，而且角度是 \(\theta/2\)（和「两个都用 \(\theta\)」有什么不同）？

回答：

代码里 psi_HF 对应 \(\lvert01\rangle\)、psi_double 对应 \(\lvert10\rangle\)，二者 正交归一。这是在 二维子空间 里做 实系数叠加 并把 范数固定为 1。

（1）为什么用 cos、sin？

两态正交归一时，最一般的 实 叠加可写成

\[ \lvert\psi\rangle = a\lvert\phi_0\rangle + b\lvert\phi_1\rangle,\quad a,b\in\mathbb{R},\quad a^2+b^2=1. \]

令 \(a=\cos\varphi,\; b=\sin\varphi\)（某一区间内的 \(\varphi\)），则 自动归一：\(\cos^2\varphi+\sin^2\varphi=1\)。

所以 cos/sin 不是物理定理「强迫」的唯一写法，而是 单位圆上最常用的参数化；等价地也可写成 \(\sqrt{1-t^2}\lvert\phi_0\rangle + t\lvert\phi_1\rangle\) 等，但和 量子门习惯 不直接对齐。

（2）为什么是 \(\theta/2\)，而不是 \(\cos\theta\) 与 \(\sin\theta\)？

这与 单量子比特绕 \(Y\) 轴的旋转 定义一致。泡利 \(Y\) 的旋转门（与 Qiskit 等常见的 \(R_y(\theta)\) 一致）为

\[ R_y(\theta) = e^{-i\theta \sigma_y/2} = \begin{pmatrix}\cos\frac{\theta}{2} & -\sin\frac{\theta}{2} \\[4pt]\sin\frac{\theta}{2} & \cos\frac{\theta}{2}\end{pmatrix}. \]

作用在计算基 \(\lvert0\rangle\) 上：

\[ R_y(\theta)\lvert0\rangle = \cos\frac{\theta}{2}\lvert0\rangle + \sin\frac{\theta}{2}\lvert1\rangle. \]

指数上是 \(\theta/2\)（李代数里生成元系数），矩阵元里就出现 \(\theta/2\) 的 cos/sin。这是 \(\mathrm{SU}(2)\) 与 Bloch 球 的标准「半角」结构：Bloch 球上转 \(\theta\)，振幅里出现 \(\theta/2\)。

因此：若你把 \(\lvert\mathrm{HF}\rangle\)、\(\lvert\mathrm{double\ exc}\rangle\) 编码成 某个比特的 \(\lvert0\rangle,\lvert1\rangle\)（或经固定幺正搬到这一对基），则 电路里一个 \(R_y(\theta)\) 制备的态 天然 就是 \(\cos(\theta/2)\lvert\mathrm{HF}\rangle+\sin(\theta/2)\lvert\mathrm{double\ exc}\rangle\)；参数 \(\theta\) 与门角度一一对应，周期、优化步长都与软件一致。

若强行写成 \(\cos\theta\lvert\mathrm{HF}\rangle+\sin\theta\lvert\mathrm{double\ exc}\rangle\)，在 \(\theta\in[0,2\pi)\) 上 振幅会扫两遍（\(\cos,\sin\) 的周期与 \(R_y\) 的 \(2\pi\) 周期 不一致），且 \(\theta\) 不再等于 电路里 \(R_y\) 的角度——容易在接 Qiskit 时混。

（3）小结

点 说明
cos、sin 归一化二维实叠加 的标准三角参数化。
\(\theta/2\) 与 \(R_y(\theta)=e^{-i\theta Y/2}\) 及 Bloch 球 半角 一致，便于 电路实现 VQE。

条目 17

日期： 2026-03-27

问题：

（Phase1 01_量子计算基础与概念映射.ipynb）为什么 VQE 演示里只用 HF 态 和 双激发态，没有单激发态？

回答：

分三层：量子化学定理（主因）、H\(_2\) 对称性与极小基（具体体系）、笔记本里的教学简化（2 比特子空间）。

（1）Brillouin 定理：闭壳层 HF 与单激发在能量上「不直接耦合」

设 \(\lvert\Phi_0\rangle\) 为 闭壳层 RHF 行列式，\(\lvert\Phi_i^a\rangle\) 表示 单激发（占据 \(i\to\) 虚轨道 \(a\)）。若轨道满足 Hartree–Fock 方程，则

\[ \langle \Phi_0 \lvert \hat{H} \rvert \Phi_i^a \rangle = 0. \]

这叫 Brillouin 定理。

因此：在 线性 CI 里，基态能量对单激发系数是 一阶退化 的——不把双激发等拉进来之前，单激发 不会 与 \(\lvert\Phi_0\rangle\) 通过 \(\hat{H}\) 发生 一阶混合 来压低能量。电子相关能的 主导修正 通常来自 双激发（MP2、CCD、CISD 里双激发的角色也与此一致）。

注意：不是说单激发行列式从希尔伯特空间里消失了，而是 \(\hat{H}\) 在 \(\lvert\Phi_0\rangle\) 与单激发块之间的矩阵元为 0，所以在「HF + 单激」张成的子空间里，能量极小值仍在 HF（一阶上）；要看到单激发的间接效应，往往要通过 与双激发的耦合（如 CISDT…）或 非 RHF 参考 等更复杂机制。

（2）H\(_2\) + STO–3G：FCI 基态几乎只在 HF + 双激子空间

同一笔记本前面用近似 FCI 波函数

\[ \lvert\psi_{\mathrm{FCI}}\rangle \approx c_{\mathrm{HF}}\lvert 1100\rangle + c_{\mathrm{double}}\lvert 0011\rangle \]

（两系数 \(\approx 0.99\) 与 \(\approx -0.11\)），刻意没写单激发构型——对 \(^1\Sigma_g^+\) 基态、闭壳层参考、极小基 而言，对称性 与 积分结构 使得 重要单激发要么系数极小，要么与参考的 \(\hat{H}\) 耦合已为 0（Brillouin），基态能量与波函数 主要由 HF +（键反键）双激发 决定。

因此：不是「物理上永远没有单激发」，而是 在这个分子 + 基组 + 对称性下，对基态而言单激发不重要；换成分子、大基组、开壳层参考，单激发会进入 ansatz（如 UCCSD 里的单激指数）。

（3）演示用的 2 量子比特、单参数 ansatz：教学上的裁剪

§6 代码把问题压到 \(\lvert01\rangle\)（记作 HF） 与 \(\lvert10\rangle\)（记作 double exc） 的 二维 子空间，用 一个 \(R_y(\theta)\) 扫能量曲线。这是为了 图解变分原理，不是要宣称 完整 H\(_2\) FCI 空间只有两维。

全空间里有更多行列式（含单激发）；选这两维是因为它们 对应前面 FCI 近似里权重最大的两个构型，且 与 Brillouin + H\(_2\) 基态物理 一致；若把单激发也纳入，至少需要 更多基向量与更多参数，图会复杂，偏离「一眼看懂 VQE」的目标。

（4）小结

原因 内容
Brillouin RHF 参考下 \(\langle\mathrm{HF}\lvert\hat{H}\rvert\text{单激}\rangle=0\)，单激 不 与 HF 直接 混合降能。
H\(_2\) 基态 极小基下基态 近似 为 HF + 主导双激；单激权重小。
笔记本设计 单参数、2 比特 演示；完整工作流见 Phase2 UCCSD 等含单激的 ansatz。

条目 18

日期： 2026-04-02

问题：

（接续 条目 17 末句）为什么说换 大分子、大基组、开壳层参考 时，单激发 往往 不能忽略，会进入 UCCSD 等 ansatz 的 单激指数？请比 H\(_2\) 极小基讲得更细。

回答：

先 纠正一个常见误读：Brillouin 定理 只保证 \(\langle\Phi_0\lvert\hat{H}\rvert\Phi_i^a\rangle=0\)（闭壳层 RHF 下 HF 与单激发块之间 矩阵元为 0），并不 保证在 完整相关计算 里 单激发系数恒为 0。下面分 大分子 / 大基组 / 开壳层 说明 为何单激会变得重要。

（1）Brillouin 之后，单激发还能从哪「长」出来？

在 CISD / CCSD 等里，决定振幅的是 整张哈密顿矩阵（或 CC 振幅方程），不只是 \(\langle\Phi_0\lvert\hat{H}\rvert\text{单激}\rangle\) 这一条。

典型链条：

  • \(\langle\Phi_0\lvert\hat{H}\rvert\text{双激}\rangle\neq 0\)（HF 与双激发 直接 耦合）；
  • \(\langle\text{单激}\lvert\hat{H}\rvert\text{双激}\rangle\) 一般 \(\neq 0\)；
  • 单激发块内部 \(\langle\Phi_i^a\lvert\hat{H}\rvert\Phi_j^b\rangle\) 也一般 非零。

因此，即便 HF↔︎单激 被 Brillouin 关掉，单激↔︎双激↔︎HF 仍可把 单激发振幅 拉成 非零（CISD 本征矢量、CCSD 的 \(T_1\) 都如此）。体系越大、基组越大，单/双激子空间的维数 与 耦合路径 越多，\(T_1\) 或 CI 单激系数 往往 越大，不能再当「永远可略」。

（2）大分子：对称性降低 + 轨道多

H\(_2\) 极小基有 很高对称（如 \(g/u\)），许多单激发 按对称性整类禁阻 或 权重极小；大分子 点群 较低（仅 \(C_1\)、\(C_s\) 等也很常见），「禁阻」变少，允许的单激发通道 激增。

同时 价层、内层、多键、配位 等使 占据–虚轨道 对数增多：同一对称块里 的 单激子空间维度 随 电子数与轨道数 增长，与双激的纠缠 更强，单激振幅 通常 显著大于 H\(_2\)–STO–3G 那种极限情形。

（3）大基组：更多虚轨道 + 「轨道松弛」信息

基组从 极小 扩到 分裂价、极化、弥散 后，虚轨道 数目与 空间分布 大变。RHF 轨道 只在平均场意义下最优；相关波函数 往往希望 占据与虚轨道之间有更细的单粒子重混合（可理解为 电子相关诱导的轨道松弛 的一部分）。在 CCSD 里，\(T_1\) 正是承担 这类单粒子重混合 的主要自由度之一（与 \(T_2\) 强耦合），故 大基组 下 \(T_1\) 通常 不可删；删成 仅 CCD / 无单激的截断 常 明显损精度。

近简并（前线轨道挤在一起）时，单激发能量分母变小 的通道增多，微扰/变分 意义上单激 更容易变大——这与 条目 1（4） 里「近前线」讨论是同一类物理。

（4）开壳层参考：Brillouin 的「闭壳层 RHF」前提常不成立

条目 17 写的 Brillouin 针对 闭壳层 RHF + 自旋限制单参考。

  • UHF / ROHF / 开壳层：参考行列式 \(\lvert\Phi_0\rangle\) 不再满足同一套 限制 Hartree–Fock 方程，常有 \(\langle\Phi_0\lvert\hat{H}\rvert\Phi_i^a\rangle\neq 0\)，单激发 可直接 与参考混合 压低能量，比闭壳层 RHF 更「表面」上就重要。
  • \(\alpha\) 与 \(\beta\) 电子 环境不同（开壳层），自旋通道不对称，许多 单激发 描述 自旋极化、自旋相关，与双激发同属必需自由度。

因此 开壳层 上 UCCSD 的 \(e^{T_1}\)（单激部分）几乎是 标准配置，而不是「可省」。

（5）与量子算法（UCCSD）的衔接

UCCSD 写 \(\lvert\psi\rangle\propto e^{T_1+T_2}\lvert\mathrm{HF}\rangle\)，\(T_1\)、\(T_2\) 对应 单、双激生成元。H\(_2\) 教学例子里 两维子空间 + 单参数 是 刻意裁剪；一般分子 上若仍 省略单激，等于 子空间不包含 CCSD 中重要的 \(T_1\) 方向，能量与性质 都会偏。大分子、大基组、开壳层 正是 \(T_1\) 变大、变不可省 的典型 regime。

一句话： Brillouin 只 挡掉 HF↔︎单激的直接矩阵元；单激仍可通过双激等间接进入波函数，且 大体系 / 大基组 放大这种耦合与 \(T_1\)；开壳层 则常使 HF↔︎单激 本身 不再为 0——故 单激发在一般量子化学与 UCCSD 里不能忽略。

问题：

（Phase1 02_Qiskit入门实践.ipynb）qc.measure([0, 1], [0, 1]) 为什么测量要写成这样？

回答：

在 Qiskit 里，QuantumCircuit.measure(qargs, cargs) 需要 两组下标，因为 量子比特 与 经典寄存器里的比特 是 两套线，必须说清楚「测谁」以及「结果存到哪」。

（1）两个列表分别是什么？

  • 第一个参数 qargs：要测量的 量子比特 编号（电路里 qubit 线的索引）。
  • 第二个参数 cargs：经典比特 编号（classical register 里存 0/1 的位置）。

按位置配对：qargs[i] 的测量结果写入 cargs[i]。

因此

qc.measure([0, 1], [0, 1])

表示：测第 0 号量子比特 → 存到经典寄存器第 0 位；测第 1 号量子比特 → 存到经典寄存器第 1 位。

两个列表 恰好都是 [0, 1] 只是「编号对齐」的常见写法，不是语法要求两边必须相同——例如你也可以（在寄存器够大的前提下）写成 qc.measure([0, 1], [3, 5])，表示结果落到经典位的第 3、5 格。

（2）为什么要分开写，而不是只写 measure([0,1])？

量子算法里 量子线 与 经典线 在电路图中是 不同横线；编译、 shots、后端读取结果时，都以 经典寄存器 为输出接口。API 设计成 显式指定 (量子源, 经典宿)，才能支持 部分测量、重排读出顺序、多量子比特映射到任意经典位 等用法。

单比特时可写 qc.measure(0, 0)（两个整数）；多比特用 等长列表。

（3）小结

写法 含义
measure([0,1], [0,1]) 量子比特 0、1 分别测到经典位 0、1。
两列表等长 必须 一一对应，否则报错。
与 [[0,1]] 混淆？ 这里是 两个 参数，各为 一维 索引列表，不是嵌套一条经典总线。

问题：

在 Qiskit 里要显式写 qc.measure([0,1], [0,1]) 指定量子比特与经典寄存器；用 PennyLane 时好像 不用 指定「存到第几个经典比特」，为什么？

回答：

这是 API 抽象层次不同，不是物理上「PennyLane 不测经典比特」。底层真机上 照样要有读出与比特串；PennyLane 在 编译到具体后端 时由 设备 / 插件 帮你把 量子线（wires） 接到 读出通道，你在前端写的是 观测量或采样接口，而不是 OpenQASM 风格的两套寄存器。

（1）Qiskit QuantumCircuit：接近「电路 IR」

QuantumCircuit 显式维护 量子寄存器 + 经典寄存器；measure(qargs, cargs) 是在电路里 真的画一条「测量并把结果写入经典线」的指令，和 OpenQASM 2 的 measure q[0] -> c[0]; 同一思路。

好处：与硬件作业、比特串 counts、部分测量顺序 对齐得 很直白；代价是你要管 经典位下标。

（2）PennyLane：「量子函数 + 观测量」抽象

你通常写的是 在哪些 wires 上 对 哪个可观测量 求值，例如：

  • qml.expval(qml.PauliZ(0))：得到 期望值（解析或有限 shots 估计）；
  • qml.sample(wires=[0, 1])：得到 采样张量；
  • qml.probs(wires=...)：概率分布。

这里 没有 用户可见的 classical_register[i]，因为 PennyLane 把电路建成 tape / QNode，再交给 device（default.qubit、lightning.qubit、IBM、Braket 等） 执行；真机路径上，插件会把 wires 映射到硬件的 物理比特与读出槽位，经典比特排列在 驱动层 处理，不暴露 成你手写第二组列表。

（3）对照小结

侧面 Qiskit（本笔记本写法） PennyLane（常见写法）
用户显式对象 量子比特线 + 经典寄存器线 wires + 测量类型（expval / sample / …）
经典位下标 自己写 measure(q, c) 一般 不写（设备封装）
典型用途 画电路、对接 Qiskit Runtime / 原始 counts 可微 VQE、混合自动微分、多后端统一接口

若你在 PennyLane 里也要 原始比特串级 的控制，仍可通过 具体设备的底层接口 或 导出成 Qiskit 电路 再 measure，那时就会重新看到「经典寄存器」这一层。

标签： #Qiskit #参数化线路 #Statevector

问题：

在 02_Qiskit入门实践.ipynb 中，下面两行各自在做什么？Qiskit 里 assign_parameters 与 Statevector.from_instruction 的作用是什么？

bound_circuit = qc_ansatz.assign_parameters({theta: theta_val})
sv_bound = Statevector.from_instruction(bound_circuit)

回答：

（1）assign_parameters({theta: theta_val})——把「符号参数」换成具体数值

  • 前面的 qc_ansatz 是用 Parameter('θ') 建出来的 参数化量子线路：门里写的是 符号 \(\theta\)，还不是某一个确定的旋转角。
  • assign_parameters 接收一个 字典（或其它 Qiskit 支持的绑定形式），键是 Parameter 对象，值是 浮点数（这里 theta_val，例如 \(\pi/4\)）。
  • 返回值 bound_circuit 是一条 已绑定 的线路：所有 \(\theta\) 都被替换成该数值，线路上不再含未定义参数，可以交给模拟器、Estimator，或再与 transpile 等配合使用。
  • 在 VQE / 变分思路里：经典优化器每给出一组参数，就 绑定一次 得到具体线路，再算能量或梯度——这一行就是 「把当前候选 \(\theta\) 写进线路」。

（2）Statevector.from_instruction(bound_circuit)——由线路得到完整量子态向量

  • Statevector 表示 归一化的态矢量（在计算基下的 \(2^n\) 维复向量，\(n\) 为量子比特数）。
  • from_instruction(...) 把 幺正部分 的线路当成从 \(\lvert 0\cdots 0\rangle\) 出发的制备：等价于先取全零初态，再按门顺序作用，得到输出态；结果封装为 Statevector 对象 sv_bound。
  • 之后可用 sv_bound[i] 读各计算基振幅、用 probabilities_dict() 看概率分布等。
  • 注意： 若线路里含 测量 或非幺正操作，from_instruction 的行为与是否支持有关；本笔记本里用于 无测量的参数化制备，语义就是 精确态模拟。

（3）两行连起来读

先 固定 \(\theta=\theta_{\mathrm{val}}\) 得到具体线路，再 在全振幅意义上模拟 该线路的输出态——即 「在参数 \(\theta_{\mathrm{val}}\) 下的波函数 \(\lvert\psi(\theta_{\mathrm{val}})\rangle\)」，便于打印主要基分量、与化学里的变分波函数类比。

问题：

（Phase1 02_Qiskit入门实践.ipynb §3.2）解释

ry(theta,0) → x(1) → cx(0,1)（原稿另有 ry(-theta,0)）在做什么？

回答：

约定：初态 为两比特 \(\lvert 00\rangle\)，Qiskit 的比特串 \(\lvert q_1 q_0\rangle\) 与 Statevector 下标一致（\(q_0\) 为 LSB）。

（1）逐步推导（前三门）

  1. ry(θ, 0) 只在 \(q_0\) 上作用：\(\lvert 0\rangle\to \cos\frac{\theta}{2}\lvert 0\rangle+\sin\frac{\theta}{2}\lvert 1\rangle\)，\(q_1\) 仍为 \(\lvert 0\rangle\)，故

\[ \lvert 00\rangle \to \cos\frac{\theta}{2}\lvert 00\rangle + \sin\frac{\theta}{2}\lvert 01\rangle. \]

  1. x(1) 把 \(q_1\) 取反：\(\lvert 0\rangle\leftrightarrow\lvert 1\rangle\)，

\[ \to \cos\frac{\theta}{2}\lvert 10\rangle + \sin\frac{\theta}{2}\lvert 11\rangle. \]

  1. cx(0,1)：\(q_0\) 为 控制，\(q_1\) 为 目标（\(q_0{=}1\) 时翻 \(q_1\)）。
    • \(\lvert 10\rangle\)：控制为 0，不变；
    • \(\lvert 11\rangle\)：控制为 1，\(\lvert 11\rangle\to\lvert 01\rangle\)。

故

\[ \lvert\psi(\theta)\rangle = \cos\frac{\theta}{2}\lvert 10\rangle + \sin\frac{\theta}{2}\lvert 01\rangle. \]

这就是 单参数下 在 \(\{\lvert 01\rangle,\lvert 10\rangle\}\) 张成的平面里转一圈（与 Phase1 里 H\(_2\) 演示用的 HF / 双激 两维子空间同一类想法）。

（2）和注释里「cos|01⟩+sin|10⟩」的关系

在 Qiskit 写法 下，cos 乘在 \(\lvert 10\rangle\)、sin 乘在 \(\lvert 01\rangle\)（见上式）。若某处写成 \(\cos\frac{\theta}{2}\lvert 01\rangle+\sin\frac{\theta}{2}\lvert 10\rangle\)，往往是 把两个比特谁先写、谁叫「第 0 位」 换了顺序，不是 数学上多出一道不同的门。对照时请以 Statevector 或 draw 输出 为准。

问题：

（Phase1 02_Qiskit入门实践.ipynb §4.4）minimize(..., method='COBYLA', options={'maxiter': 100, 'rhobeg': 0.5}) 在 Qiskit 里是怎么实现的？

回答：

这一格 并不是 Qiskit 实现的，而是 scipy.optimize.minimize（文件里 from scipy.optimize import minimize）。Qiskit 只负责 StatevectorEstimator 算能量；经典优化器 这里用的是 SciPy 自带的 COBYLA。

（1）COBYLA 是什么？

COBYLA（Constrained Optimization BY Linear Approximations）是 M. J. D. Powell 提出的 无导数（只用函数值）优化算法，适合 噪声目标、黑箱代价（与 VQE 里「测期望值」很搭）。SciPy 通过 Fortran 实现的 cobyla 接口（或后续维护版本）接到 Python；算法细节在 SciPy 文档 / Powell 原文，不在 Qiskit 核心库里。

（2）options 里两项什么意思？

选项 含义（COBYLA）
maxiter 允许的最大 迭代次数（一次迭代里会多次调用 cost_function，故 函数评估次数 可能大于 maxiter）。
rhobeg 初始 信赖域 / 步长尺度（变量空间里的初始「试探半径」量级）；参数维数、典型尺度不同时可调，影响前几步走多远。

具体语义以 当前 SciPy 版本 的 minimize COBYLA 说明 为准。

（3）若要用「Qiskit 自带」优化器呢？

qiskit-algorithms 里的 VQE 等可把 COBYLA、SLSQP、L_BFGS_B 等放在 qiskit_algorithms.optimizers（多是对 SciPy 或其它库的 封装）。逻辑仍是：量子端算 ⟨H⟩，经典端跑优化；本教学格为透明起见 直接调 SciPy。
