# 可视化与布局调整说明

以后若要改**力导向图手感**、**节点大小/颜色**、**侧栏公式**或**界面文案**，按下面文件定位即可。

---

## 1. 力导向图（弹簧松紧、拖拽手感）

**文件：** `src/components/QuantumGraph.jsx`

在 `useEffect` 里创建 `d3.forceSimulation` 的那段代码中，主要改这些：

### 1.1 连线「弹簧」——`force('link', ...)`

| 含义 | 代码位置 | 调参建议 |
|------|----------|----------|
| **理想边长**（越大图越「松」、节点间距越大） | `.distance((e) => { ... const dist = { root: …, module: …, … } })` | 改 `dist` 里各类型数值；最后一行 `return dist[srcType] \|\| 155` 是未匹配类型时的默认距离 |
| **弹簧刚度**（越小越容易拖开、越「软」） | `.strength((e) => (e.type === 'cross' ? 0.03 : 0.12))` | 普通边用第二个数；`e.type === 'cross'` 是跨模块虚线，一般用更小值 |

当前示例（以仓库内代码为准）：

- 距离：`root / module / category / topic / leaf` 各一档，外加默认 `155`
- 刚度：虚线约 `0.03`，普通边约 `0.12`

### 1.2 节点互斥——`force('charge', ...)`

| 含义 | 代码 | 说明 |
|------|------|------|
| 库仑斥力 | `d3.forceManyBody().strength((d) => { const s = { root: …, … }; return s[d.type] \|\| … })` | 负数越大（绝对值越大）节点越互相推开；整体太挤可略加大绝对值，太散可略减小 |

### 1.3 碰撞体积——`force('collision', ...)`

| 含义 | 代码 | 说明 |
|------|------|------|
| 碰撞半径 | `.radius((d) => getTypeConfig(d.type).radius + 36)` | `+` 后面的数越大，节点之间「硬边界」越大，越不容易叠在一起 |
| 碰撞力强度 | `.strength(0.72)` | `0~1`，越大碰撞越「硬」 |

### 1.4 仿真阻尼与收敛

| 含义 | 代码 | 说明 |
|------|------|------|
| 速度衰减 | `.velocityDecay(0.35)` | 越大越快停住、抖动少 |
| alpha 衰减 | `.alphaDecay(0.022)` | 越大仿真越快「冷静」下来 |

### 1.5 拖拽节点时

在 `d3.drag()` 的 `'start'` 里：

- `simulation.alphaTarget(0.55)`：拖拽时给仿真「加油」，数值越大跟手程度通常越好（可试 `0.4`～`0.65`）
- `event.sourceEvent?.stopPropagation?.()`：避免和画布缩放/平移抢事件

### 1.6 缩放与节点冲突

`d3.zoom()` 上的 `.filter(...)`：

- 在 `g.node` 上的 `mousedown` / `touchstart` / `dblclick` 返回 `false`，避免在节点上触发画布平移或双击缩放，方便单独拖节点、双击聚焦。

### 1.7 节点视觉大小（圆半径）

同一文件里绘制节点时用 `getTypeConfig(d.type).radius`，**数值定义不在本文件**，而在下面的 `initialGraph.js`。

## 2. 节点半径、颜色、发光（图谱语义尺寸）

**文件：** `src/data/initialGraph.js`

导出对象 **`NODE_TYPE_CONFIG`**：

```js
export const NODE_TYPE_CONFIG = {
  root:     { color: '#...', radius: 36, glowColor: '...' },
  module:   { color: '#...', radius: 28, ... },
  category: { color: '#...', radius: 22, ... },
  topic:    { color: '#...', radius: 16, ... },
  leaf:     { color: '#...', radius: 12, ... },
}
```

- 改 **`radius`**：力导向里碰撞、标签位置、圆的大小都会跟着变（与 `QuantumGraph.jsx` 里碰撞 `+ 36` 等叠加）。
- 改 **`color` / `glowColor`**：节点描边与光晕。

节点**文案与结构**（标题、描述、Markdown 正文、边）在同文件的 **`INITIAL_GRAPH`** 里改。

## 3. 侧栏 Markdown 与数学公式

| 需求 | 文件 |
|------|------|
| 启用/关闭公式、换 KaTeX 选项 | `src/components/NodePanel.jsx` — `md.use(mdKatex, { ... })` |
| 全局 KaTeX 样式表 | `src/main.jsx` — `import 'katex/dist/katex.min.css'` |
| 侧栏内公式字号、块级公式滚动 | `src/styles/global.css` — `.markdown-body .katex` / `.katex-display` |
| 依赖版本 | `package.json` — `katex`、`markdown-it-katex` |

节点正文里使用 **`$...$`（行内）**、**`$$...$$`（独立行）** 写 LaTeX（与 `markdown-it-katex` 规则一致）。

## 5. 快速对照表

| 你想改… | 优先打开的文件 |
|---------|------------------|
| 弹簧更紧/更松、更好拖、间距 | `src/components/QuantumGraph.jsx` |
| 节点圆大小、颜色 | `src/data/initialGraph.js` → `NODE_TYPE_CONFIG` |
| 知识树内容与链接 | `src/data/initialGraph.js` → `INITIAL_GRAPH` |
| 侧栏公式显示 | `NodePanel.jsx` + `global.css` + `main.jsx` |
| 界面固定文案 | `src/App.jsx` |
